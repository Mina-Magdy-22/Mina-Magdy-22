### 🚗 Used Cars Price Prediction
> **Machine Learning Project, Kaggle**

A full ML pipeline to build a reliable price-prediction baseline for used cars, with strong emphasis on **preventing data leakage** and providing a reproducible, transparent workflow.

**Dataset:** `Autos.csv`, vehicle attributes (brand, model, yearOfRegistration, powerPS, kilometer, fuelType, gearbox, notRepairedDamage, price, etc.) | **Target:** `price`

| Stage | What I Did | Tools Used |
|---|---|---|
| 🔎 Data Audit | Types, missingness, unrealistic values (yearOfRegistration, powerPS) | Pandas, NumPy |
| 🧹 Cleaning | Dropped irrelevant columns, winsorized extreme powerPS, removed invalid/zero prices | Pandas |
| 🔧 Imputation | Mode for categorical; KNN/SimpleImputer for numeric | Scikit-Learn |
| ⚙️ Feature Engineering | Age, Power_per_Age, LOOCV Brand_Model_Value (row-excluded) | Python, Pandas |
| 🔡 Encoding | One-hot for low-cardinality; frequency/LOOCV encodings for grouped features | Scikit-Learn |
| 🤖 Modeling | Pipeline-based Linear Regression & Random Forest with cross-validation | Scikit-Learn |
| 📊 Diagnostics | Residual plots, Durbin–Watson, Spearman correlations, OLS summary | Statsmodels, Matplotlib |
| 🛡️ Leakage Checks | Checked price-leaking columns & near-perfect correlations with target | Custom logic |

**📈 Key Results:**

| Model | R² Score | MAE |
|---|---|---|
| Linear Regression (baseline) | ≈ 0.55 | ≈ 1,844 |
| **Random Forest (best)** | **≈ 0.875** | - |

> ⚠️ Diagnostics indicate heteroscedastic residuals - robust losses or target transforms recommended for production.

**✅ Engineering Best Practices Applied:**
- LOOCV brand/model encoding computed excluding the current row (CV-safe)
- Imputation, encoding & scaling inside `Pipeline` / `ColumnTransformer`
- Time-aware split recommendations documented for temporal data
- Fully documented limitations (heteroscedasticity, geographic/temporal biases)
